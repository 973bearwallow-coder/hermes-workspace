# Paw Prints — Divi Rebuild Decision Checklist

Use this checklist to determine when a full Divi rebuild becomes the right call.

## Trigger Conditions (any one = time to reconsider)

- [ ] **Custom functionality needed** — Require WordPress plugins or custom post types that can't be achieved with direct file edits.
- [ ] **Multi-developer handoff** — Another developer needs to maintain the site and requires a standard CMS workflow.
- [ ] **Site complexity exceeds direct edits** — Too many pages/components to manage without a CMS.
- [ ] **Major redesign** — Ground-up visual/structural overhaul justified.
- [ ] **Performance issues** — Current approach causing slow load times or technical debt.
- [ ] **Client/SEO requirements** — Need WordPress-specific SEO plugins (Yoast, RankMath) or structured data tools.
- [ ] **E-commerce or booking integration** — Need WooCommerce, Amelia, or similar WP-native booking systems.
- [ ] **Tom decides to hand off day-to-day editing** — If Tom wants to make changes himself without Atlas in the loop.
